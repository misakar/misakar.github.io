用C实现协程

什么是协程

协程, 英文是coroutine或green thread, 从这两个称呼上可以一窥协程的特点: 协作式调度(coroutine), 低机器级开销(green).
协程是"人为的"对线程执行流的分割. 所谓人为, 即CPU和操作系统并不需要管理和调度协程, CPU只需要处理线程的切换, 协程的切换由用户级程序完成, 而且多个协程共享单线程内存, 所以协程是低机器级开销; 此外, 因为是人为切换, 所以协程并不需要竞争CPU时间, 例如: 当consumer协程需要数据的时候, 直接从当前线程的执行栈退出, 把执行权让给productor协程, productor生成数据后再把执行权还给consumer, 当下一次consumer需要数据时, productor则从退出时的执行点继续向下执行生成新的数据. 上述过程的伪代码如下:

def productor():
    for data in data_list:
        # 生成新数据
        write_data_to_buffer(data, buffer)
        await consumer # yield productor; resume consumer

def consumer():
    # data是data_list中的新数据, 而不是上一次生成的数据
    data = read_data_from_buffer(buffer)  
    await productor # yield consumer; resume productor

productor()

如何实现协程

从上面的分析和伪代码可以看出, 实现协程的关键在于协程的切换, 即multiple execution stack. 协程的切换可以简单的理解为栈间的"跳跃". 所以要想实现协程, 就需要实现保存、恢复以及调度每个执行栈.
